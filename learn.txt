Prerequisites: Open Docker

Before running any commands, make sure Docker is running on your laptop:

* **Windows/Mac:** Open the **Docker Desktop** application from your apps folder and wait until the little status icon in the bottom corner turns green ("Running").
* **Linux:** Ensure your docker daemon is active by running `sudo systemctl start docker`.

---

### The 10-Step Terminal Command Routine

Open your terminal (Mac/Linux) or Command Prompt/PowerShell (Windows) and execute these exact 10 steps.

1. **Navigate to Your Project Folder:** Command 1.
Change directory directly into the existing folder on your hard drive where your programming game project files are saved:

```bash
cd "C:\Users\YOUR_NAME\path\to\your\devops-game-engine"

```

*(If you are on a Mac, use your forward slash path formatting: `cd "/Users/YOUR_NAME/path/to/your/devops-game-engine"`)*


2. **Initialize Offline Git Tracking:** Command 2.
Turn your existing game project directory into a local, offline Git repository so Jenkins can read its revision history:

```bash
git init

```


3. **Stage Existing Project Files:** Command 3.
Gather all your current project files, your Java code, `pom.xml`, and your configuration files into Git's preparation staging area:

```bash
git add .

```


4. **Commit the Project Snapshot:** Command 4.
Save a local version snapshot of your game engine code directly into your computer's hard drive branch repository:

```bash
git commit -m "baseline: offline source code ready for automation"

```


5. **Copy the Absolute File Path:** Command 5.
Print the exact, absolute path of your current folder location. **Copy the text output this command prints**, you will need it in the web interface later:

```bash
pwd

```

*(On Windows Command Prompt, type `cd` instead of `pwd` to print your absolute current directory path).*


6. **Launch Jenkins using Docker:** Command 6.
Run this command to download and start Jenkins inside an isolated container wrapper. This maps Jenkins to port `8082` and grants it access to your laptop's underlying Docker socket:

```bash
docker run -d -p 8082:8080 -p 50000:50000 -v jenkins_home:/var/jenkins_home -v /var/run/docker.sock:/var/run/docker.sock --name jenkins-lab jenkins/jenkins:lts

```


7. **Retrieve the Secret Security Key:** Command 7.
Fetch the initial administrator setup password generated by the Jenkins container. **Copy the long random string of characters** it spits out near the bottom of the logs:

```bash
docker logs jenkins-lab

```


8. **Grant Docker Clearance (Part 1):** Command 8.
Instruct the container to assign root ownership to the mounted Docker execution socket file so Jenkins can trigger local compilation containers:

```bash
docker exec -u 0 -it jenkins-lab chown root:root /var/run/docker.sock

```


9. **Grant Docker Clearance (Part 2):** Command 9.
Open up read and write permissions on the Docker socket inside the pipeline workspace container environment to bypass safety blockades:

```bash
docker exec -u 0 -it jenkins-lab chmod 666 /var/run/docker.sock

```


10. **Verify App Deployment After Web Setup:** Command 10.
*(Execute this command ONLY after completing the Web Interface Setup below)*. Once your pipeline completes its execution run, verify your game service container is successfully running live on your machine:

```bash
docker ps --filter name=game-instance

```


---

### 🌐 The Web Interface Setup (Between Steps 9 & 10)

Once you execute **Command 9**, open up your web browser and configure the dashboard options:

1. Navigate in your browser to: `http://localhost:8082`
2. Paste the password you copied from **Command 7** into the box and select **Install Suggested Plugins**. Create your admin user profile.
3. On the main dashboard, click **New Item** $\rightarrow$ Name it `local-game-pipeline` $\rightarrow$ Select **Pipeline** $\rightarrow$ Click **OK**.
4. Scroll to the bottom **Pipeline** block, change the **Definition** dropdown to **Pipeline script from SCM**.
5. Set **SCM** to **Git**.
6. In **Repository URL**, type `file://` immediately followed by the exact path you copied from **Command 5**:
* *Example:* `file:///C:/Users/YOUR_NAME/path/to/your/devops-game-engine`


7. Click **Save**.

---

### 🚀 Running the Pipeline Practice

Click **Build Now** on the left side of your screen. Jenkins will instantly read your local project code straight from your hard drive, compile your backend code, build a clean runtime Docker image, drop old iterations, and boot your game application live at `http://localhost:8080/health` with zero internet lag!
